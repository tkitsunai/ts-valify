# TDD(Test-Driven Development) Guideline Skill

本Skillは、実装フェーズにおいてTDDで開発を進めるための手順を定めています。実装フェーズでは本Skillに従ってユニットテストを書きながらRed->Green->Refactorサイクルで実装を進めること。

Refactorの際には、コードの可読性や保守性を高めるためのリファクタリングを行うことが求められます。Greenの後には、コードの品質を向上させるためのリファクタリングを行うことが推奨されます。必要に応じてリファクタリングに必要なスキルを読み込むこと。

## TDDサイクル

このスキルではTDDを以下の内容をルールとして定義し守ることを求める。

- **テストを先に書く**: 実装コードを先に書いてからテストを後付けしてはならない
- **テストの失敗を確認する**: テストを書いたら該当テストが必ず実行し、期待通りの失敗をすることを確認する
- **小さなステップで進む**: 一度に大量のコードを書かず、テスト→実装のサイクルを細かく回す
- **リファクタリングを行う**: テストが成功した後、コードの品質を向上させるためのリファクタリングを行う

## Agentsの行動指針

### やってはいけないこと

- テストを実行せずに実装をしてはならない
